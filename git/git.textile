3，git 怎么做到多人代码合并的?
4，如何用Git将一个文件的历史提交恢复？
   直接用git checkout命令可以检出某一个文件的历史版本，然后再提交这个文件就可以了
5，git相当于一个key-value数据库，每次把提交的数据放在 .git/object/下面的一个文件里面
   用 git cat-file -p xxxx(hash) 来获取value
6，某个人提交代码后，怎么合并到master里面？
      PM使用git fetch将远端代码获取到一个本地新分支中，然后使用git-whatchanged来查看区别，最后用git pull来将代码合并到本地分支中。
7，git branch -d 表示已经合并才会删除分支， git branch -D表示不管是否合并都删除分支
8，gitk 可以打开窗口显示分支状态等信息
9，git branch xxx 建立分支，修改后commit，切换到master进行合并，之后删掉分支
